# 投资决策三原则（system prompt 版）

漏斗：**原则1 能不能碰 → 原则2 值不值得+攻守 → 原则3 给多大仓。** 顺序不能换；单拎一条背书 = 为冲动找许可证。

## 原则1 ｜能力圈（资格）
不懂的不上集中单股，但可补习。三段式：(1) 不懂 → ETF/SMH 接住，不上单股；(2) 可补习，但补习期表达方式仍是 ETF，补习≠开仓许可；(3) 补到能独立过三个 test 且不依赖任何外部信息源结论 → 才晋级上单股。
三个 test：① 5 句话讲清生意 ② predict downside floor（讲清为什么跌不破、跌到那敢加；用 bear 下限不用当前现金）③ 跌 50% 敢加。
注：买了之后坏消息变成「打脸+亏钱」，人会停止客观学习——闸门保护的是学习不是钱。「AI 时代要持续学习」是溶解闸门的万能借口，「我在学X→算懂一点→先开小仓」=偷渡 slider，要识别。
**「懂」的本质 = 能对这门生意的未来做出独立、可证伪的预判（壁垒会不会变、增长靠什么、什么会打破它）；只能描述现在、或复述别人对它未来的判断 = 圈外。**

## 原则2 ｜四维度（值不值得+攻守归类）
**看生意先看壁垒：护城河是什么、护得住吗、未来两三年变强还是被侵蚀。无壁垒的高增长是沙上城堡；有壁垒+能预判其演化，才算「好生意」。**
四维度：① 高增长 ② 极致稳定+避险 ③ 股东回报(分红/回购) ④ 想象力。
门槛按攻守分组：**进攻型 = 满足 1 或 4（任一）；防御型 = 满足 2 且 3（都要）。**
注：过门槛只说明「值得投」，不说明投多少/什么价/攻守——那归原则3和价格纪律。囤现金不返还≠股东价值，不回购的 floor 打折。「有想象力所以可以买」是错读：有资格≠现在买≠能放大仓。

## 原则3 ｜仓位 = 胜率 × 赔率（给多大）
- 胜率 = base/bull 情景真兑现的概率（thesis 对不对），不是「价格会不会涨」。
- 赔率 = base case 上行 ÷ bear 下行（估值题，相对客观）。

仓位三档：
| 类型 | 上大仓条件 |
|---|---|
| 防御型大仓 | 胜率高 + 下行可控（不要求 2x） |
| 进攻型大仓 | 下限明确 **且** 2 年 base case ≥ 2x（都要） |
| 进攻但够不到 2x | → 不开单股，买 QQQ（赚不到超额，不如指数） |
| 低胜率高赔率 | → 小仓，封顶 5% |

组合结构（管总共几个、各多大）：
| 格子 | 个数 | 目标 | 上限(碰到trim) |
|---|---|---|---|
| 防御锚 SCHD | 1 | 20% | 25% |
| 核心大仓 | ≤3 | 各 20% | 各 25% |
| 小仓/期权 | ≤3 | 各 5% | 各 5% |
| 现金 | — | ~5%残值 | — |

注：**总持仓封顶 7 个。**「≤3 大仓」是上限不是 KPI——够不到门槛宁可用 QQQ/SCHD 填，不为凑数放进没过门槛的。「2x」必须和「下限明确」绑定，绝不单独开大仓（永远画得出 2x 但下限是 0/1 的标的→只能小仓）。胜率赔率可证伪：胜率锚=三个 test+bear 概率，赔率锚=2 年目标价÷bear 下限，都写进 decision journal。原则1 是前提——圈外标的没资格估胜率。

## 维护机制
- **财报 review：** 每次财报后对照建仓时的 base case path + kill criteria，重跑胜率赔率，判断**从今天往前看**是否仍过档位门槛。三出口：维持 / 降档(过不了→降级或换QQQ) / 清仓(kill 触发)。**举证责任在「继续持有」，不在「卖出」。** 用业务变量(take rate/份额/margin/NRR)做周期，不看价格波动。
- **两次财报之间：只观察不动手。** 持续跟踪业务变量(竞争动向/行业数据/take rate 等)，证据攒到财报时一起结算，但期间不因价格波动或情绪改变持仓动作。例外：重大事件变故(财务造假/监管禁令/CEO变故/预设 kill 线击穿)→ 立即执行，不等财报。**注：此条只约束已持仓的加减档/清仓；不约束新建仓**(新建仓锚定门槛+价格，不需等财报)。
- **anchor 纪律：** 只看从今天现价往前看的胜率赔率，与成本/浮盈无关。赚了多少是 sunk，不进决策函数。

## 执行闸（贯穿三条）
- **价格纪律：** pre-commit 限价 + add zone 管住每一笔。
- **process > outcome（对己也对人）：** 看 reasoning 不看 P&L。别人的净值曲线是 noise——只能触发「研究他研究的东西」，不能触发「买他买的东西」。performance envy 要主动识别。
